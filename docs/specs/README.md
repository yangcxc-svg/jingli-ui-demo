## 当前项目状态判断

当前项目已经完成 Demo 级核心闭环：

- 用户可以在前端输入送礼需求
- 后端可以调用 AI 能力并流式返回文本
- 后端可以返回商品卡片
- 商品可以加入礼单
- 购物车页面可以展示礼单并调整数量

但项目仍有明显演示态特征：

- 前端主文件过大，页面和组件耦合严重
- 推荐、礼单、组合方案的领域边界还不清晰
- 礼单是内存版，后端重启会丢失
- 商品知识库还是种子数据
- 缺少测试、部署和协作约束

因此后续完善应优先稳定核心体验，再做架构抽象和工程化。
